# ⚡ Ускоритель мобильного интернета в России 
## 📋 Что делает парсер

`parser.py` — автоматизированный инструмент на Python, который:

1. **Скачивает VLESS-конфиги** из открытых источников, перечисленных в `sources.txt` (поддерживаются тысячи источников на GitHub и других платформах).

2. **Очищает и валидирует** найденные конфиги:
   - Проверяет корректность UUID и структуры VLESS-URL.
   - Удаляет дубликаты (по содержимому URL без учёта имени конфига).

3. **Фильтрует по SNI и домену**: оставляет только конфиги, чей SNI/Host совпадает с доменами российских CDN и сервисов из `domain.txt` (или `whitelist.txt`, если он есть). Это гарантирует, что конфиги работают при использовании белых списков.

4. **Переименовывает конфиги** для удобства: в имя включается протокол (WS, gRPC, TCP, Reality и т.д.) и название сервиса (например, «VK», «Яндекс», «Wildberries»). Добавляется метка `[#РКП]`.

5. **Кодирует специальные символы** в URL для совместимости с разными VPN-клиентами.

6. **Проверяет работоспособность** каждого конфига через ядро [Xray-core](https://github.com/XTLS/Xray-core):
   - Xray скачивается автоматически при первом запуске.
   - Для каждого конфига поднимается временный SOCKS5-прокси и делается тестовый HTTP-запрос.
   - Дополнительно проверяется TCP-подключение и TLS-рукопожатие.
   - Нерабочие конфиги исключаются из итогового списка.

7. **Сохраняет рабочие конфиги** в файл `configs/url_work.txt`, отсортированные по пингу (от быстрых к медленным).

### Поддерживаемые протоколы

| Протокол | Описание |
|---|---|
| **VLESS + Reality** | Мимикрия под реальный TLS-трафик |
| **VLESS + TLS** | Классический зашифрованный туннель |
| **VLESS + WS** | WebSocket поверх TLS |
| **VLESS + gRPC** | gRPC поверх TLS |
| **VLESS + XHTTP** | HTTP Upgrade / XHTTP |
| **VLESS + H2** | HTTP/2 |

### Используемые файлы

| Файл | Назначение |
|---|---|
| `sources.txt` | Список URL-источников с VLESS-конфигами |
| `domain.txt` | Список доменов для фильтрации по SNI |
| `iP.txt` | Список IP-адресов российских CDN |
| `whitelist.txt` | (опционально) Пользовательский список доменов для фильтрации |
| `configs/url_work.txt` | Итоговый список рабочих конфигов (обновляется автоматически) |

## 🚀 Как запустить парсер

### Требования
- Windows 10/11 (64-bit) — Xray загружается только для Windows
- Python 3.9 или новее
- Интернет-соединение

### Установка зависимостей
```bash
pip install -r requirements.txt
```

### Запуск

**Непрерывный режим** (запускает циклы каждый час):
```bash
python parser.py
```

**Однократный запуск** (один цикл и завершение):
```bash
python parser.py --once
```

### Настройки

В начале файла `parser.py` можно изменить параметры:

| Параметр | По умолчанию | Описание |
|---|---|---|
| `THREADS_DOWNLOAD` | `50` | Параллельных потоков для скачивания |
| `XRAY_MAX_WORKERS` | `30` | Параллельных потоков для Xray-тестирования |
| `XRAY_TIMEOUT` | `3` | Таймаут теста конфига (секунды) |
| `CYCLE_DELAY` | `3600` | Пауза между циклами в непрерывном режиме (секунды) |

## 🤖 Автоматическое обновление на GitHub

Репозиторий настроен на **автоматический запуск парсера через GitHub Actions** каждые 6 часов.

Рабочий процесс (`.github/workflows/update_configs.yml`):
1. Запускается на Windows-раннере GitHub.
2. Устанавливает Python и зависимости.
3. Выполняет парсер в режиме `--once`.
4. Копирует результат в `configs/url_work.txt`.
5. Создаёт git-коммит и пушит обновление в репозиторий.

Вы также можете запустить обновление вручную через вкладку **Actions** → **Обновление конфигов VPN** → **Run workflow**.

## ✍️ Справка по использованию конфигов

1. Конфиги взяты из открытых источников и отфильтрованы под работу при **белых списках** (split tunneling).
2. Конфиги переименованы для удобства: в имени указан протокол и сервис, через который идёт трафик.
3. **ИСПОЛЬЗУЙТЕ КОНФИГУРАЦИИ ТОЛЬКО ПРИ БЕЛЫХ СПИСКАХ** — это снижает нагрузку и позволяет большему числу пользователей подключаться.
4. При включённом VPN не заходите в российские сервисы (ВКонтакте, Сбербанк и т.п.) — CDN-домен конфига может быть заблокирован Роскомнадзором по вашей вине.
5. Если конфиги не работают — попробуйте другой VPN-клиент или другие конфиги из подборки.
6. Количество рабочих конфигов постоянно меняется из-за обновлений у поставщиков.
